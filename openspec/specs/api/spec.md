### Requirement: Middlewareuser

The API SHALL support `MIDDLEWARE requireAuth(req: Request, res: Response, next: NextFunction)` to validates the presence and validity of a jwt bearer token in the authorization header. if valid, attaches the user id and role to the request object for downstream use.

**Request:**

```json
{
  "headers": {
    "authorization": "Bearer <JWT>"
  }
}
```

**Response:**

```json
{
  "success": {
    "status": 200,
    "description": "Proceeds to the next middleware/route handler."
  },
  "error": {
    "status": 401,
    "body": {
      "error": "string (e.g., 'Missing authorization header' or 'Invalid or expired token')"
    }
  }
}
```

#### Scenario: ValidTokenProvided
- **GIVEN** A valid JWT is provided in the Authorization header.
- **WHEN** The requireAuth middleware is invoked.
- **THEN** The user ID and role are attached to the request object, and the next middleware/route handler is
called.

#### Scenario: MissingAuthorizationHeader
- **GIVEN** No Authorization header is provided.
- **WHEN** The requireAuth middleware is invoked.
- **THEN** A 401 response is returned with the error message 'Missing authorization header'.

#### Scenario: InvalidOrExpiredToken
- **GIVEN** An invalid or expired JWT is provided in the Authorization header.
- **WHEN** The requireAuth middleware is invoked.
- **THEN** A 401 response is returned with the error message 'Invalid or expired token'.

### Requirement: Middlewareuser

The API SHALL support `MIDDLEWARE requireRole(role: string)(req: Request, res: Response, next: NextFunction)` to validates that the authenticated user has the specified role. must be used after requireauth middleware.

**Request:**

```json
{
  "headers": {
    "authorization": "Bearer <JWT>"
  },
  "internal": {
    "userRole": "string (attached to request by requireAuth)"
  }
}
```

**Response:**

```json
{
  "success": {
    "status": 200,
    "description": "Proceeds to the next middleware/route handler."
  },
  "error": {
    "status": 403,
    "body": {
      "error": "string (e.g., 'Insufficient permissions')"
    }
  }
}
```

#### Scenario: UserHasRequiredRole
- **GIVEN** The user has the required role (e.g., 'admin').
- **WHEN** The requireRole middleware is invoked with the role 'admin'.
- **THEN** The next middleware/route handler is called.

#### Scenario: UserDoesNotHaveRequiredRole
- **GIVEN** The user does not have the required role (e.g., user role is 'user' but 'admin' is required).
- **WHEN** The requireRole middleware is invoked with the role 'admin'.
- **THEN** A 403 response is returned with the error message 'Insufficient permissions'.
