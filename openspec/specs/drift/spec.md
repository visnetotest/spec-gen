## Purpose

[PARTIAL SPEC — file too large to fully analyze (2 parts)] Provides git diff integration for drift
detection by determining what files changed between the current working tree and a base reference.

## Entities

### DriftIssue

Represents a drift issue detected between code and specifications.

**Properties:**

| Name | Type | Description |
|------|------|-------------|
| id | string | Unique identifier for the drift issue |
| kind | DriftIssueKind | Type of drift issue |
| severity | DriftSeverity | Severity level of the drift issue |
| message | string | Description of the drift issue |
| filePath | string | Path to the file with the drift issue |
| domain | string | Domain related to the drift issue |
| specPath | string | Path to the specification file |
| changedLines | object | Lines added and removed in the file |
| suggestion | string | Suggestion to resolve the drift issue |

**Relationships:**

- belongs to Project (Drift issue belongs to a project)
- belongs to File (Drift issue is related to a file)
- belongs to OpenSpecDomain (Drift issue is related to an OpenSpec domain)
