## Purpose

Detects discrepancies between code changes and existing specifications, identifying gaps, stale
specs, uncovered files, and orphaned specs to maintain alignment between code and documentation.

## Requirements

### Requirement: UserValidation

The system SHALL validate User according to these rules:
- Email must be unique across all users.
- Password must be hashed before storage using bcrypt with a cost factor of 12.
- User ID must be a non-empty string.
- Role must be one of the predefined system roles.

#### Scenario: UserRegistrationWithUniqueEmail
- **GIVEN** A new user provides a unique email, password, and name.
- **WHEN** The register method is called with these details.
- **THEN** A new user is created, and an authentication token is returned.

#### Scenario: UserRegistrationWithDuplicateEmail
- **GIVEN** A user provides an email that is already registered.
- **WHEN** The register method is called with these details.
- **THEN** An error is thrown indicating the email is already registered.

#### Scenario: SuccessfulUserLogin
- **GIVEN** A registered user provides the correct email and password.
- **WHEN** The login method is called with these credentials.
- **THEN** An authentication token is returned.

#### Scenario: FailedUserLoginWithIncorrectPassword
- **GIVEN** A registered user provides the correct email but incorrect password.
- **WHEN** The login method is called with these credentials.
- **THEN** An error is thrown indicating invalid credentials.

#### Scenario: FailedUserLoginWithNonExistentEmail
- **GIVEN** A non-registered email is provided.
- **WHEN** The login method is called with this email and any password.
- **THEN** An error is thrown indicating the user was not found.

### Requirement: AuthResultValidation

The system SHALL validate AuthResult according to these rules:
- Token must be a valid JWT signed with the system's secret key.
- Token must expire after 24 hours (86400 seconds).
- User ID must match the ID of an existing User.

#### Scenario: TokenVerificationSuccess
- **GIVEN** A valid JWT token is issued for a user.
- **WHEN** The verifyToken method is called with this token.
- **THEN** The method returns the user ID and role embedded in the token.

#### Scenario: TokenVerificationFailure
- **GIVEN** An invalid or expired JWT token is provided.
- **WHEN** The verifyToken method is called with this token.
- **THEN** The method returns null.
