# Validator Specification

> Generated by spec-gen v1.0.0 on 2026-02-28 21:35

## Purpose

Validates OpenSpec specifications against OpenSpec standards, project-specific rules, and best
practices. Provides actionable feedback for compliance.

## Requirements

### Requirement: Validateagainstopenspec

The system SHALL validate an openspec object against the openspec standard (e.g., openapi 3.x).

#### Scenario: ValidationFailsOnInvalidSchemaReference
- **GIVEN** An OpenSpec object with a path referencing a non-existent schema.
- **WHEN** validateAgainstOpenSpec is called with the spec.
- **THEN** The validation result indicates failure with an error about the invalid schema reference.

### Requirement: Validateheadinghierarchy

The system SHALL validate the heading hierarchy in markdown descriptions within the openspec object (e.g., no skipped heading levels).

#### Scenario: ValidationFailsOnSkippedHeadingLevel
- **GIVEN** An OpenSpec object with a Markdown description containing a level 3 heading following a level 1
heading (skipping level 2).
- **WHEN** validateHeadingHierarchy is called with the spec.
- **THEN** The validation result indicates failure with an error about the skipped heading level.

### Requirement: Validatecustomrules

The system SHALL validate the openspec object against project-specific custom rules (e.g., naming conventions, required metadata).

#### Scenario: ValidationFailsOnCustomNamingRuleViolation
- **GIVEN** An OpenSpec object with a schema named 'user_model' and a custom rule requiring PascalCase naming.
- **WHEN** validateCustomRules is called with the spec and custom rules.
- **THEN** The validation result indicates failure with an error about the naming convention violation.
