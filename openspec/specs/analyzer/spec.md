## Purpose

Analyzes the structural dependencies between files in a codebase to identify metrics like
centrality, clusters, cycles, and importance scores.

## Entities

### RepositoryMap

A comprehensive representation of a code repository's structure, content, and analysis results,
combining file metadata, significance scoring, and domain-specific clustering.

**Properties:**

| Name | Type | Description |
|------|------|-------------|
| metadata | RepositoryMapMetadata | Core metadata about the repository and analysis process. |
| summary | RepositoryMapSummary | High-level statistics and aggregated data about the repository. |
| highValueFiles | ScoredFile[] | Files identified as having high business or architectural significance. |
| entryPoints | ScoredFile[] | Files that serve as entry points to the application (e.g., main files, app roots). |
| schemaFiles | ScoredFile[] | Files containing data schemas, models, or type definitions. |
| configFiles | ScoredFile[] | Configuration files for the project or its dependencies. |
| clusters | FileClusters | Files grouped by directory, domain, or architectural layer. |
| allFiles | ScoredFile[] | All files analyzed in the repository. |

**Relationships:**

- composition RepositoryMapMetadata (Contains metadata about the repository and analysis process.)
- composition RepositoryMapSummary (Contains aggregated statistics and high-level data about the repository.)
- composition FileClusters (Contains files grouped by directory, domain, or architectural layer.)
- aggregation ScoredFile (Contains references to files with significance scores and metadata.)

### DependencyNode

Represents a file in the dependency graph with its import/export relationships and calculated
metrics.

**Properties:**

| Name | Type | Description |
|------|------|-------------|
| id | string | Unique identifier for the node, typically the absolute file path. |
| file | ScoredFile | The file associated with this node, containing metadata and scores. |
| exports | ExportInfo[] | List of exports from this file. |
| metrics | object | Calculated metrics for this node in the dependency graph. |

**Relationships:**

- has many DependencyEdge (A DependencyNode can have multiple incoming and outgoing DependencyEdges representing import/export relationships.)
- many-to-one FileCluster (A DependencyNode may belong to a FileCluster representing a group of related files.)

### DependencyEdge

Represents an import relationship between two files in the dependency graph.

**Properties:**

| Name | Type | Description |
|------|------|-------------|
| source | string | The id of the source DependencyNode (file importing). |
| target | string | The id of the target DependencyNode (file being imported). |
| importedNames | string[] | List of names imported from the target file. |
| isTypeOnly | boolean | Indicates if the import is type-only (e.g., TypeScript types). |
| weight | number | Weight of the edge, used for calculating metrics (e.g., 0.5 for type-only imports, 1 otherwise). |

**Relationships:**

- many-to-one DependencyNode (A DependencyEdge connects exactly one source DependencyNode to one target DependencyNode.)

### FileCluster

Represents a group of related files that form a cohesive module or domain within the project.

**Properties:**

| Name | Type | Description |
|------|------|-------------|
| id | string | Unique identifier for the cluster. |
| name | string | Human-readable name for the cluster, often derived from common directory or domain. |
| files | string[] | List of file ids (DependencyNode ids) that belong to this cluster. |
| internalEdges | number | Number of edges (imports) between files within this cluster. |
| externalEdges | number | Number of edges (imports) from files in this cluster to files outside the cluster. |
| cohesion | number | Metric indicating how tightly coupled the files within the cluster are (higher is better). |
| coupling | number | Metric indicating how much the cluster depends on other clusters (lower is better). |
| suggestedDomain | string | Suggested domain or module name for this cluster based on file paths or content. |

**Relationships:**

- has many DependencyNode (A FileCluster contains multiple DependencyNodes (files).)

**Relationships:**

- composition GraphNode (Contains multiple GraphNode instances representing files in the codebase.)

## Requirements

### Requirement: RepositoryMapValidation

The system SHALL validate RepositoryMap according to these rules:
- RepositoryMap must include metadata with projectName, projectType, rootPath, analyzedAt, and version.
- RepositoryMap must include a summary with totalFiles, analyzedFiles, skippedFiles, languages, frameworks, and directories.
- highValueFiles, entryPoints, schemaFiles, and configFiles must be subsets of allFiles.
- File clusters must accurately reflect the directory structure and domain/layer classifications.

#### Scenario: RepositoryMapGeneration
- **GIVEN** A valid repository path and analysis configuration
- **WHEN** The repository mapping process is initiated
- **THEN** A RepositoryMap is generated with metadata, summary, and file clusters
- **AND** All files are scored and categorized by significance and domain

#### Scenario: HighValueFileIdentification
- **GIVEN** A repository with files of varying significance
- **WHEN** The significance scoring is applied
- **THEN** Files with high business or architectural value are identified in highValueFiles
- **AND** These files are prioritized for further analysis

### Requirement: RepositoryMapMetadataValidation

The system SHALL validate RepositoryMapMetadata according to these rules:
- projectName must be a non-empty string.
- projectType must be a valid value from the ProjectType enum.
- rootPath must be an absolute path to an existing directory.
- analyzedAt must be a valid ISO 8601 timestamp.
- version must follow semantic versioning (e.g., '1.0.0').

#### Scenario: MetadataValidation
- **GIVEN** A repository map is being generated
- **WHEN** Metadata is populated
- **THEN** All required fields are present and valid
- **AND** The metadata provides accurate context for the analysis

### Requirement: RepositoryMapSummaryValidation

The system SHALL validate RepositoryMapSummary according to these rules:
- totalFiles must be a non-negative integer.
- analyzedFiles must be a non-negative integer and less than or equal to totalFiles.
- skippedFiles must be a non-negative integer and equal to totalFiles - analyzedFiles.
- languages must include at least one entry if totalFiles > 0.
- The sum of fileCount in languages must equal totalFiles.

#### Scenario: SummaryGeneration
- **GIVEN** A repository with files of various languages and frameworks
- **WHEN** The repository is analyzed
- **THEN** A summary is generated with accurate file counts and language breakdowns
- **AND** Detected frameworks are listed with their confidence levels

### Requirement: LanguageBreakdownValidation

The system SHALL validate LanguageBreakdown according to these rules:
- language must be a non-empty string.
- extension must start with a dot (e.g., '.ts').
- fileCount must be a non-negative integer.
- percentage must be a number between 0 and 100.

#### Scenario: ValidLanguageBreakdownAccepted
- **GIVEN** A valid LanguageBreakdown object with all required fields
- **WHEN** The object is validated
- **THEN** Validation passes with no errors

### Requirement: TechnologyDetectionValidation

The system SHALL validate TechnologyDetection according to these rules:
- name must be a non-empty string
- category must be one of 'database', 'auth', or other valid categories
- detected must be a boolean
- confidence must be one of 'high', 'medium', or 'low'
- evidence must be an array of non-empty strings

#### Scenario: DetectPostgresqlWithPgDependency
- **GIVEN** A project with 'pg' in dependencies
- **WHEN** The analyzer runs detection
- **THEN** A TechnologyDetection entity is created with name 'PostgreSQL', category 'database', detected true,
and evidence including 'PostgreSQL client in dependencies'

#### Scenario: DetectMongodbWithMongooseDependency
- **GIVEN** A project with 'mongoose' in dependencies
- **WHEN** The analyzer runs detection
- **THEN** A TechnologyDetection entity is created with name 'MongoDB', category 'database', detected true, and
evidence including 'MongoDB client in dependencies'

#### Scenario: DetectJwtAuthWithJsonwebtokenDependency
- **GIVEN** A project with 'jsonwebtoken' in dependencies
- **WHEN** The analyzer runs detection
- **THEN** A TechnologyDetection entity is created with name 'JWT Auth', category 'auth', detected true, and
evidence including 'JWT library in dependencies'

#### Scenario: NoDetectionWhenDependencyIsMissing
- **GIVEN** A project without any relevant dependencies for a technology
- **WHEN** The analyzer runs detection
- **THEN** A TechnologyDetection entity is created with detected false and empty evidence

### Requirement: DependencyNodeValidation

The system SHALL validate DependencyNode according to these rules:
- id must be a non-empty string and unique across all DependencyNodes.
- file must be a valid ScoredFile object.
- exports must be an array of ExportInfo objects.
- metrics must contain valid numeric values for inDegree, outDegree, betweenness, and pageRank.

#### Scenario: NodeCreationWithValidFile
- **GIVEN** A valid ScoredFile and its FileAnalysis.
- **WHEN** The DependencyGraphBuilder.build method is called.
- **THEN** A DependencyNode is created with correct id, file, exports, and initial metrics.

#### Scenario: NodeMetricsCalculation
- **GIVEN** A DependencyGraph with nodes and edges.
- **WHEN** The calculateDegrees, calculateBetweenness, and calculatePageRank methods are called.
- **THEN** Each DependencyNode's metrics are updated with accurate inDegree, outDegree, betweenness, and
pageRank values.

### Requirement: DependencyEdgeValidation

The system SHALL validate DependencyEdge according to these rules:
- source and target must be non-empty strings and correspond to existing DependencyNode ids.
- importedNames must be an array of non-empty strings.
- weight must be a positive number.

#### Scenario: EdgeCreationForValidImport
- **GIVEN** A valid import relationship between two files.
- **WHEN** The DependencyGraphBuilder.buildEdges method is called.
- **THEN** A DependencyEdge is created with correct source, target, importedNames, isTypeOnly, and weight.

#### Scenario: EdgeWeightAdjustmentForTypeOnlyImports
- **GIVEN** A type-only import between two files.
- **WHEN** The DependencyGraphBuilder.buildEdges method processes the import.
- **THEN** The resulting DependencyEdge has a weight of 0.5.

### Requirement: FileClusterValidation

The system SHALL validate FileCluster according to these rules:
- id must be a non-empty string and unique across all FileClusters.
- files must be an array of non-empty strings corresponding to existing DependencyNode ids.
- internalEdges and externalEdges must be non-negative integers.
- cohesion and coupling must be non-negative numbers.

#### Scenario: ClusterDetectionWithHighCohesion
- **GIVEN** A set of files with many internal imports and few external imports.
- **WHEN** The DependencyGraphBuilder.detectClusters method is called.
- **THEN** A FileCluster is created with high cohesion and low coupling values.

#### Scenario: ClusterNamingBasedOnFilePaths
- **GIVEN** A cluster of files located in the same directory.
- **WHEN** The DependencyGraphBuilder.detectClusters method processes the files.
- **THEN** The resulting FileCluster has a name derived from the common directory path.

### Requirement: DependencyGraphResultValidation

The system SHALL validate DependencyGraphResult according to these rules:
- nodes must be an array of valid DependencyNode objects.
- edges must be an array of valid DependencyEdge objects.
- clusters must be an array of valid FileCluster objects.
- metrics must contain valid numeric values for all properties.

#### Scenario: CompleteGraphAnalysis
- **GIVEN** A set of analyzed files with their import/export relationships.
- **WHEN** The DependencyGraphBuilder.build method completes.
- **THEN** A DependencyGraphResult is created with all nodes, edges, clusters, and metrics.

### Requirement: DependencyGraphValidation

The system SHALL validate DependencyGraph according to these rules:
- Nodes must have unique identifiers.
- Edges must reference existing nodes.
- Graph must not contain duplicate edges between the same source and target nodes.
- Options must include valid dampingFactor (0 < dampingFactor < 1) and maxIterations (maxIterations > 0).

#### Scenario: CalculateBetweennessCentrality
- **GIVEN** A dependency graph with nodes and edges
- **WHEN** The calculateBetweenness method is called
- **THEN** Each node's betweenness metric is updated based on shortest-path calculations

#### Scenario: CalculatePagerankScores
- **GIVEN** A dependency graph with nodes and edges
- **WHEN** The calculatePageRank method is called
- **THEN** Each node's pageRank metric is updated based on iterative convergence

#### Scenario: DetectClustersInTheGraph
- **GIVEN** A dependency graph with nodes grouped by directory
- **WHEN** The detectClusters method is called
- **THEN** FileCluster instances are returned with cohesion and coupling metrics

#### Scenario: DetectCyclesInDependencies
- **GIVEN** A dependency graph with circular references
- **WHEN** The detectCycles method is called
- **THEN** All unique cycles in the graph are identified and returned

### Requirement: GraphNodeValidation

The system SHALL validate GraphNode according to these rules:
- Node ID must be unique within the graph.
- File metadata must include valid name and directory properties.
- Metrics must be initialized with default values (0 for betweenness and pageRank).

#### Scenario: UpdateNodeMetrics
- **GIVEN** A GraphNode instance with default metrics
- **WHEN** Dependency analysis algorithms complete
- **THEN** The node's metrics are updated with calculated values for betweenness and pageRank

### Requirement: GraphEdgeValidation

The system SHALL validate GraphEdge according to these rules:
- Source and target must reference existing nodes in the graph.
- Edge must be unique (no duplicate edges between the same source and target).

#### Scenario: ValidateEdgeReferences
- **GIVEN** A GraphEdge instance with source and target IDs
- **WHEN** The edge is added to the graph
- **THEN** The graph verifies that both source and target nodes exist

### Requirement: ImportInfoValidation

The system SHALL validate ImportInfo according to these rules:
- source must be a non-empty string
- isRelative, isPackage, and isBuiltin must be mutually exclusive (only one can be true)
- importedNames must be an array of strings
- line must be a positive integer

#### Scenario: ParseADefaultImport
- **GIVEN** A source file with the statement 'import X from "module"'
- **WHEN** The file is parsed
- **THEN** An ImportInfo entity is created with hasDefault=true and importedNames=['X']

#### Scenario: ParseANamespaceImport
- **GIVEN** A source file with the statement 'import * as X from "module"'
- **WHEN** The file is parsed
- **THEN** An ImportInfo entity is created with hasNamespace=true and importedNames=['X']

#### Scenario: IdentifyARelativeImport
- **GIVEN** A source file with the statement 'import X from "./local"'
- **WHEN** The file is parsed
- **THEN** An ImportInfo entity is created with isRelative=true

#### Scenario: IdentifyABuiltinImport
- **GIVEN** A source file with the statement 'import fs from "fs"'
- **WHEN** The file is parsed
- **THEN** An ImportInfo entity is created with isBuiltin=true

### Requirement: ExportInfoValidation

The system SHALL validate ExportInfo according to these rules:
- name must be a non-empty string
- kind must be one of: 'function', 'class', 'variable', 'type', 'interface', 'enum', 'unknown'
- line must be a positive integer
- reExportSource must be a non-empty string if isReExport is true

#### Scenario: ParseADefaultExport
- **GIVEN** A source file with the statement 'export default function X() {}'
- **WHEN** The file is parsed
- **THEN** An ExportInfo entity is created with isDefault=true, name='X', and kind='function'

#### Scenario: ParseANamedExport
- **GIVEN** A source file with the statement 'export const X = 1'
- **WHEN** The file is parsed
- **THEN** An ExportInfo entity is created with isDefault=false, name='X', and kind='variable'

#### Scenario: ParseAReExport
- **GIVEN** A source file with the statement 'export { X } from "module"'
- **WHEN** The file is parsed
- **THEN** An ExportInfo entity is created with isReExport=true and reExportSource='module'

### Requirement: FileAnalysisValidation

#### Scenario: HandleParsingErrors
